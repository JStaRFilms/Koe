# Builder Prompt Instructions

## Mandatory Mockup-Driven Implementation
The `docs/mockups` folder is the **UNQUESTIONABLE source of truth** for all front-end UI/UX.

You must NOT deviate from the layout, color palette, typography, or component structure defined in the mockups.
Before implementing any component or page:
1. Open the corresponding `.html` mockup file (e.g., `docs/mockups/home.html`) using the `view_file` tool.
2. Carefully read the HTML structure, the Tailwind utility classes utilized, and the inline CSS variables/animations if any.
3. Replicate the styling exactly in your React/Next.js components. 

Do not rely on generic generic generic shadcn blocks if they contradict the mockup structure. The provided mockups already encapsulate the correct UI/UX Pro Max aesthetic constraints.

*Design with intention. Code with precision.*
