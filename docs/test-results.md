### Security Issues (From Audit Task 04)

#### Issue 1: Hardcoded Encryption Key
**Severity:** Critical  
**Location:** `src/main/services/settings.js:6`  
**Description:** Encryption key is hardcoded as `koe-super-secret-key-replace-in-prod`  
**Impact:** Anyone with source/binary access can decrypt all user data including Groq API keys  
**Fix:** Generate unique key per installation using `crypto.randomBytes(32)`

#### Issue 2: XSS in History Panel
**Severity:** Critical  
**Location:** `src/renderer/components/history-panel.js:66-78`  
**Description:** User transcription text inserted directly into innerHTML without sanitization  
**Impact:** Malicious HTML/JS in transcriptions will execute. Could steal API keys.  
**Fix:** Use textContent instead of innerHTML

#### Issue 3: Missing CSP Headers
**Severity:** High  
**Location:** HTML files (`index.html`, `settings-window.html`)  
**Description:** No Content Security Policy meta tags  
**Fix:** Add CSP restricting scripts to self

#### Issue 4: No Input Validation
**Severity:** High  
**Location:** `src/main/ipc.js`  
**Description:** IPC handlers accept user input without validation  
**Fix:** Validate all IPC inputs before processing
