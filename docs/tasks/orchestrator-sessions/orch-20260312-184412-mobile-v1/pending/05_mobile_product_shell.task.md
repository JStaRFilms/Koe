## Objective

Turn the functional mobile pipeline into a real product shell with recorder UX, clipboard completion, history, settings, onboarding, and launch affordances.

## Scope

**In scope:**
- recorder UI states
- copy-to-clipboard completion flow
- history view
- settings view
- onboarding / first-run instructions
- usage visibility
- launch shortcuts or deep-link entry into the recorder screen if the platform setup is ready

**Out of scope:**
- Android IME
- paid accounts
- team/cloud sync

## UX Requirements

### Recorder Experience

- Clear primary record control
- Visible states for:
  - idle
  - listening/recording
  - processing
  - copied
  - warning/error
- Show enough session feedback that the user trusts what happened

### Completion Flow

- On successful completion, copy the final transcript to the clipboard automatically
- Show explicit confirmation that the text is ready to paste elsewhere
- Provide a fast retry or copy-again action

### History And Settings

- History must show recent transcripts and retry/copy affordances
- Settings must include:
  - Groq API key
  - language
  - prompt style
  - model choice if retained from desktop behavior
  - enhancement toggle
- First-run onboarding must explain that mobile is clipboard-first, not system keyboard dictation

## Technical Requirements

- Persist settings and history locally
- Use secure storage for the API key and normal local storage/SQLite/structured storage for non-secret state
- Reuse shared-core types and state contracts where appropriate
- Keep UI code distinct from recording/provider code
