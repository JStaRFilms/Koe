## 📋 Objective

Fix the production build so that VAD (Voice Activity Detection) assets load correctly, allowing transcription to work.

## 🎯 Root Cause Identified

In `src/renderer/audio/vad.js`, the VAD is configured with:
```javascript
baseAssetPath: '/vad/',
onnxWASMBasePath: '/vad/',
```

**Problem:** These are absolute paths. In production:
- App loads via `file://` protocol from `dist/renderer/index.html`
- `/vad/` resolves to `file:///vad/` (doesn't exist)
- Actual assets are at `dist/renderer/vad/` (relative to HTML file)

**In Dev:** The custom Vite plugin `vadAssetsPlugin()` intercepts `/vad/*` requests and serves from `node_modules/` - but this plugin's `configureServer` hook only runs in dev mode.

## 🛠️ Fixes Required

### Fix 1: Change VAD Paths to Relative
**File:** `src/renderer/audio/vad.js`

Change:
```javascript
baseAssetPath: '/vad/',
onnxWASMBasePath: '/vad/',
```

To:
```javascript
baseAssetPath: './vad/',
onnxWASMBasePath: './vad/',
```

### Fix 2: Add Missing WASM Files to public/vad/
The `public/vad/` folder only has ONNX files. In production, the VAD library also needs:
- `ort-wasm-simd-threaded.mjs` (from `node_modules/onnxruntime-web/dist/`)
- `ort-wasm-simd-threaded.wasm` (from `node_modules/onnxruntime-web/dist/`)
- `vad.worklet.bundle.min.js` (from `node_modules/@ricky0123/vad-web/dist/`)

**Action:** Copy these files to `public/vad/` so they're available in production.

### Fix 3: Verify viteStaticCopy Configuration
The `vite.config.js` has `viteStaticCopy` that should copy these files, but verify:
- The glob patterns work correctly
- Files end up in `dist/renderer/vad/`
