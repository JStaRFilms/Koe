# [FR-024] Android IME / Keyboard Integration

## Labels
`Future`, `enhancement`, `android`, `ime`, `planned`

## User Story
As an Android user, I want Koe available directly from the keyboard layer, so I can use it while typing across apps.

## Proposed Solution

Build Koe as an Android **voice-input IME** rather than trying to inject Koe into Samsung Keyboard, Gboard, or other third-party keyboards.

The host app should remain the control center for onboarding, settings, API key entry, and history. The IME should provide a minimal voice-first keyboard surface that:
- lets the user start and stop dictation from the keyboard layer
- transcribes and refines audio using Koe's existing service model where practical
- inserts final text into the focused field
- falls back to clipboard if direct insertion is rejected

Initial scope should stay intentionally narrow:
- Android only
- voice-first input surface
- no full QWERTY buildout
- no Quick Settings tile work in this feature
