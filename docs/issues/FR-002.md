# [FR-002] Mic Audio Capture

## Labels
`MUS`, `enhancement`, `audio-pipeline`

## User Story
As a user, I want my mic audio captured at 16kHz, so Whisper gets optimal input.

## Proposed Solution

### Overview
Use `navigator.mediaDevices.getUserMedia()` in the renderer to capture mic audio. Process through an AudioWorklet for real-time PCM extraction at 16kHz mono.

### Implementation Flow
1. Request mic permission via `getUserMedia({ audio: { sampleRate: 16000, channelCount: 1 } })`
2. Create `AudioContext` with 16kHz sample rate
3. Connect `MediaStreamSource` → `AudioWorkletNode` for PCM extraction
4. AudioWorklet posts Float32Array chunks to main thread via `port.postMessage`
5. Accumulate PCM samples in buffer for WAV encoding

### Key Considerations
- 16kHz mono is optimal for Whisper — don't capture at 48kHz and downsample
- AudioWorklet runs in a separate thread — no main thread blocking
- Handle permission denied gracefully (show error UI)
- Clean up streams and contexts when recording stops

## Acceptance Criteria

- [ ] Audio captured from default microphone
- [ ] Sample rate is 16kHz mono
- [ ] Uses AudioWorklet (not deprecated ScriptProcessorNode)
- [ ] Handles mic permission denied with user-friendly error
- [ ] Audio stream properly cleaned up when recording stops
