# [FR-010] Settings: API Key

## Labels
`MUS`, `enhancement`, `settings`

## User Story
As a user, I want to enter my Groq API key securely, so the app can call the API.

## Proposed Solution

### Overview
Settings panel with API key input field. Key is encrypted via electron-store's `encryptionKey` option. Validate the key by making a test API call.

### Implementation Flow
1. Settings panel UI: labeled input with show/hide toggle, "Save" button
2. On save: encrypt and store via electron-store
3. Validate key: make a lightweight API call to Groq (e.g., models list)
4. Show validation status: ✓ Valid / ✗ Invalid
5. If no key saved: show onboarding prompt on first launch

### Key Considerations
- API key must never be logged or sent to any service other than Groq
- Use `electron-store` with `encryptionKey` for at-rest encryption
- Renderer never sees the raw key — all API calls happen in main process
- Consider a "Test Connection" button

## Acceptance Criteria

- [ ] API key input field in settings panel
- [ ] Key is stored encrypted via electron-store
- [ ] Key can be validated with test API call
- [ ] Validation result shown to user (valid/invalid)
- [ ] First-launch experience prompts for API key
