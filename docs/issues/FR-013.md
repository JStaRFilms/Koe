# [FR-013] Auto-Paste

## Labels
`SHOULD`, `enhancement`, `system-integration`

## User Story
As a user, I want text auto-pasted into my previously-focused app, so I don't have to Ctrl+V manually.

## Proposed Solution

### Overview
After copying text to clipboard, simulate a Ctrl+V keystroke in the previously-focused application. Save the focused window reference before Koe activates.

### Implementation Flow
1. Before activating Koe window, store the currently focused window
2. After transcription is copied to clipboard, restore focus to previous window
3. Simulate Ctrl+V keypress via `@nut-tree/nut-js` or Electron's `webContents.sendInputEvent`
4. This is opt-in (off by default in settings)

### Key Considerations
- Some apps block programmatic paste (e.g., secure fields)
- Default to clipboard-only — auto-paste is opt-in
- May need slight delay between focus restore and paste simulation
- Windows-specific: use Win32 API or nut-js for reliable paste

## Acceptance Criteria

- [ ] Text is pasted into previously-focused app when enabled
- [ ] Feature is opt-in (disabled by default)
- [ ] Falls back gracefully if paste simulation fails
- [ ] Setting persists across restarts
