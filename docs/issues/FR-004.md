# [FR-004] Audio Chunking

## Labels
`MUS`, `enhancement`, `audio-pipeline`

## User Story
As a user, I want speech buffered into ~3-5s WAV chunks, so they fit Groq's rate limits naturally.

## Proposed Solution

### Overview
When VAD detects speech end, the accumulated PCM buffer is encoded as a WAV file and sent for transcription. If speech continues beyond ~5s, split into chunks at natural pause points.

### Implementation Flow
1. Accumulate PCM Float32Array samples during speech
2. On speech end from VAD: finalize chunk
3. If speech duration > 5s without pause: force-split at 5s boundary
4. Encode each chunk as WAV (PCM 16-bit, 16kHz, mono)
5. Send WAV blob to main process via IPC for Groq API call

### Key Considerations
- WAV header must be valid (44 bytes, correct format fields)
- Chunk size stays well under Groq's 25MB limit (~3-5s at 16kHz = ~96-160KB)
- Overlapping chunks could improve transcription quality but adds complexity — skip for V1
- Track audio seconds per chunk for rate limit accounting

## Acceptance Criteria

- [ ] Speech segments encoded as valid WAV files
- [ ] WAV is PCM 16-bit, 16kHz, mono
- [ ] Chunks are ~3-5 seconds
- [ ] Long speeches are split at reasonable boundaries
- [ ] Audio length in seconds is tracked per chunk
