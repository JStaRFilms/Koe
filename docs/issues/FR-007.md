# [FR-007] Floating Transcription Window

## Labels
`MUS`, `enhancement`, `ui`

## User Story
As a user, I want a compact, always-on-top window showing live text, so I can see what's being transcribed.

## Proposed Solution

### Overview
Create a frameless, transparent, always-on-top Electron BrowserWindow with glassmorphism styling. Display live transcription text as it arrives. Make the window draggable and resizable.

### Implementation Flow
1. Create `BrowserWindow` with `alwaysOnTop: true`, `frame: false`, `transparent: true`
2. Render a compact UI: recording indicator, text area, minimal controls
3. Use glassmorphism CSS: `backdrop-filter: blur(20px)`, semi-transparent dark background
4. Update text content via IPC as transcription results arrive
5. Make window draggable via custom title bar region (`-webkit-app-region: drag`)

### Key Considerations
- Window should not steal focus from the user's active app
- Window size should be small by default (~400x200) but resizable
- Text should auto-scroll as new transcription arrives
- Recording state indicator (pulsing dot / waveform animation)
- Use `frontend-design` and `ui-ux-pro-max` skills for premium UI

## Acceptance Criteria

- [ ] Window is always on top of other windows
- [ ] Window is frameless with glassmorphism design
- [ ] Window is draggable
- [ ] Live text updates as transcription arrives
- [ ] Recording state is visually indicated (pulse/animation)
- [ ] Window does not steal focus from other apps
