# [FR-003] Voice Activity Detection

## Labels
`MUS`, `enhancement`, `audio-pipeline`

## User Story
As a user, I want VAD to detect when I'm speaking and when I stop, so only speech is transcribed.

## Proposed Solution

### Overview
Integrate `@ricky0123/vad-web` (Silero VAD) to detect speech start/end events. Use these events to control audio buffering — only buffer audio during speech segments.

### Implementation Flow
1. Install `@ricky0123/vad-web` via npm
2. Initialize VAD with Silero ONNX model
3. Feed raw audio from AudioWorklet into VAD
4. On `speech_start`: begin buffering PCM
5. On `speech_end`: finalize buffer, trigger chunking + transcription
6. Allow configurable silence threshold (how long silence = end of speech)

### Key Considerations
- Silero VAD needs ONNX runtime — ensure it's bundled correctly with Vite
- VAD sensitivity may need tuning — expose threshold in settings (future)
- Must handle edge case: user pauses mid-sentence (short silence ≠ end of speech)
- Model files need to be accessible in the packaged app

## Acceptance Criteria

- [ ] Silero VAD correctly detects speech start and end
- [ ] Audio is only buffered during speech segments
- [ ] Background noise does not trigger false positives (reasonable threshold)
- [ ] VAD model loads successfully in packaged Electron app
