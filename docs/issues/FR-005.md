# [FR-005] Groq Whisper Transcription

## Labels
`MUS`, `enhancement`, `groq-integration`

## User Story
As a user, I want audio sent to Groq Whisper and text returned, so I get accurate transcriptions.

## Proposed Solution

### Overview
Send WAV audio chunks to Groq's Whisper API endpoint. Parse the transcription response and relay text back to the renderer.

### Implementation Flow
1. Receive WAV buffer from renderer via IPC
2. Create `FormData` with `file` (WAV blob), `model` ("whisper-large-v3-turbo"), optional `language`
3. POST to `https://api.groq.com/openai/v1/audio/transcriptions`
4. Parse response `{ text: "..." }`
5. Send transcribed text back to renderer via IPC
6. Handle errors: 429 (rate limit), 401 (bad key), network errors

### Technical Approach

```javascript
// src/main/services/groq.js
async function transcribe(wavBuffer, language = 'auto') {
  const formData = new FormData();
  formData.append('file', new Blob([wavBuffer], { type: 'audio/wav' }), 'audio.wav');
  formData.append('model', 'whisper-large-v3-turbo');
  if (language !== 'auto') formData.append('language', language);

  const response = await fetch('https://api.groq.com/openai/v1/audio/transcriptions', {
    method: 'POST',
    headers: { 'Authorization': `Bearer ${apiKey}` },
    body: formData
  });
  return (await response.json()).text;
}
```

### Key Considerations
- API key must be read from encrypted electron-store
- FormData in Node.js may need `node-fetch` or `undici` — check Electron's built-in fetch
- Handle empty transcriptions (silence detected as speech)
- Log transcription latency for debugging

## Acceptance Criteria

- [ ] WAV chunks are sent to Groq Whisper API
- [ ] Transcribed text is returned and displayed in UI
- [ ] API errors are caught and shown to user (not silent)
- [ ] 429 rate limit responses trigger retry/queue behavior
- [ ] Empty transcriptions are handled gracefully
